# 阶段五十七：小资金灰度候选计划生成

Stage57 属于 Stage53-60：灰度前真实数据 / QMT dry-run / 小资金灰度准备，不属于 Stage61-75 UI 产品化阶段。本阶段不提前开发 UI，但继续检查完整 Stage1-75 工程阶段计划和 Stage61-75 UI 产品化计划是否仍被保留。

## 安全边界

- Stage57 是小资金灰度候选计划生成。
- Stage57 不等于实盘授权。
- Stage57 只生成候选计划材料，用于未来人工复核和灰度前审批。
- Stage57 不调用 `xttrader`，不导入 `XtQuantTrader`。
- Stage57 不真实下单。
- Stage57 不查询资金、持仓、订单、成交。
- Stage57 不真实发送通知。
- Stage57 不自动 approve，不绕过 Risk Gate，不绕过 Human Approval。
- `READY_FOR_GRAY_CANDIDATE_REVIEW` 只表示小资金灰度候选计划材料可供人工复核，不代表实盘授权。

## 候选计划覆盖范围

候选计划覆盖小资金灰度资金上限、单笔交易上限、单日交易上限、ETF 白名单、最大持仓数、最大单标的仓位、组合最大仓位、现金保留比例、最大回撤触发阈值、单日亏损触发阈值、数据质量阻断条件、Risk Gate 阻断条件、Human Approval 人工审批流程、回滚流程、熔断流程、停止灰度条件、复盘报告要求和下一阶段人工审批包准备计划。

## 默认保守限制

默认候选资金上限为 1000.00，单笔上限为 200.00，单日上限为 300.00，最大持仓数为 2，最大单标的仓位为 20%，组合最大仓位为 50%，现金保留比例为 50%，单日亏损触发阈值为 1%，最大回撤触发阈值为 3%，最大下单次数为 2，停止后冷却期为 3 天。以上仅用于计划材料，不能用于真实下单。

## Roadmap / UI 产品化路线复核

本阶段继续检查完整 Stage1-75 工程阶段计划和 Stage61-75 UI 产品化计划。UI 不直接访问 QMT，UI 不能绕过 Risk Gate，UI 不能绕过 Human Approval，UI 不能自动 approve。本阶段不提前开始 UI 开发。

## 下一阶段预告

Stage58 建议为“阶段五十八：小资金灰度前最终人工审批包”。Stage58 仍不能直接实盘，只能做小资金灰度前最终人工审批包；不调用 `xttrader`，不查询真实资金、持仓、订单、成交，不下单，不真实通知。
